# Спецификация продукта: Единая цифровая экосистема здоровья

## 1. Описание проекта
Разработка комплексной мобильной платформы для управления здоровым образом жизни. Продукт заменяет разрозненные инструменты, объединяя в себе функции персонального умного тренера, диетолога и платформы для связи с профильными специалистами. Основа системы — алгоритмы искусственного интеллекта (ИИ), обеспечивающие персонализацию тренировок (дома, на улице, в зале) и контроль питания.

## 2. Цели и задачи
### Для пользователей:
* Формирование здоровых привычек и отказ от вредных через пошаговое достижение целей (снижение веса, набор мышечной массы, поддержание формы).
* Снижение когнитивной нагрузки: система берет на себя напоминания о занятиях, контроль водного баланса и автоматический расчет калорий, белков, жиров и углеводов (КБЖУ).

### Для бизнеса:
* Обеспечение высокого показателя удержания пользователей за счет глубокой персонализации и минимального ручного ввода.
* Масштабирование на международные рынки.
* **Монетизация (Гибридная модель):**
  * Подписка на расширенные функции платформы.
  * Нативная встроенная реклама.
  * Комиссионные сборы с платных онлайн-консультаций реальных специалистов.

## 3. Архитектура и основной функционал

### 3.1. Глубокая адаптация и персонализация
* **Анкетирование:** На старте собирается матрица из 23 параметров (включая рост, вес, качество сна, наличие инвентаря, медицинские ограничения).
* **Автоматика:** Моментальный расчет индекса массы тела и суточной нормы потребления нутриентов.
* **Интерфейс знакомства:** Плавный переход между экранами с наглядным индикатором прогресса текущего шага и ненавязчивой кнопкой «Пропустить».

### 3.2. Тестовое занятие и калибровка ИИ
* Пользователю предлагается короткая 10-минутная профильная тренировка.
* По итогам собирается обратная связь (оценка сложности), на основе которой алгоритм окончательно калибрует долгосрочный план нагрузок.

### 3.3. Модуль физической активности
* **Умная генерация:** ИИ создает расписание и состав тренировок.
* **Ручное управление:** Доступ к библиотеке упражнений с возможностью изменения интервалов отдыха и количества подходов.
* **Медиа и связь:** Встроенный видеопроигрыватель для демонстрации техники. Под видео интегрирован модуль текстовых сообщений для быстрой связи с закрепленным наставником.

### 3.4. Умный сканер питания
* Использование камеры смартфона в связке с нейросетями для распознавания блюд по фотографии.
* Автоматическое определение объема порции, примерной калорийности и раскладки по макронутриентам с выводом данных поверх изображения.

### 3.5. Отслеживание прогресса
* **Визуализация данных:** Динамические графики изменения веса и физических показателей (проекция результатов).
* **Фотодневник:** Сравнение состояния формы «До и После».
* **Сенсорная интеграция:** Фоновая синхронизация с системными агрегаторами (HealthKit, Google Fit) для учета шагов и активности.

### 3.6. Рабочее пространство специалиста
* Отдельная панель управления (веб-версия).
* **Возможности:** Ведение клиентской базы, аудит дневников питания, общение в чатах и конструирование авторских программ тренировок.

## 4. Требования к интерфейсу и опыту взаимодействия (UI/UX)
Визуальная концепция строится на строгом, «дорогом» минимализме, опираясь на принципы Material Design и актуальные тенденции.

* **Цветовая схема и элементы:** За основу берется сочетание глубокого фиолетового и розового оттенков. Карточки контента располагаются на чистом белом (для светлой темы) или затемненном (для темной темы) фоне для идеальной читаемости. Кнопки целевых действий — крупные (от 52px в высоту) для комфортного взаимодействия на ходу.
* **Динамические интерфейсы (Ситуативный дизайн):** Отказ от статичных экранов. Система формирует графики и блоки данных «на лету» под текущий контекст пользователя.
* **Принцип нулевого ввода:** Максимальное использование датчиков устройства (биометрия, геолокация, шагомер) для автоматического заполнения данных без участия человека.
* **Органичный дизайн:** Проектирование «зон цифрового отдыха». Интерфейс не перегружается предупреждениями, использует природные формы и скругления (по макету), создавая спокойную психологическую среду.
* **Навигация:** Все ключевые разделы (главная, тренировки, камера сканера, профиль) закреплены в нижней панели для удобного управления одной рукой. Адаптация под системные режимы энергосбережения и фокусировки.

## 5. Технологический стек платформы
Инфраструктура строится на современных решениях, обеспечивающих высокую производительность и кроссплатформенность:

### Клиентская часть (Мобильное приложение)
* **Фреймворк:** React Native + TypeScript (строгая типизация).
* **Среда разработки:** Expo (Управляемый рабочий процесс) с маршрутизацией на базе файлов (Expo Router).
* **Стилизация:** NativeWind (подход на основе служебных классов) с методологией атомарного дизайна.
* **Анимации и списки:** React Native Reanimated, Shopify FlashList, React Native Bottom Sheet.
* **Работа с данными:** TanStack Query для управления состояниями, кэширования и обеспечения работы в условиях отсутствия сети (Оптимистичный интерфейс).
* **Иконки:** Lucide React Native.

### Серверная часть и база данных
* **Инфраструктура:** Supabase (Аутентификация пользователей, PostgreSQL, Файловое хранилище, Граничные функции для интеграции ИИ).
* **Рабочее пространство тренера:** Next.js (предпочтительно, для связи с единой базой).

### Интеграции и DevOps
* **Платежный шлюз:** API ЮKassa и система вебхуков для обработки подписок.
* **Сборка и публикация:** EAS (Expo Application Services) в связке с GitHub Actions для непрерывной интеграции.
