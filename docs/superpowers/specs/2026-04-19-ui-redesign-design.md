# Fitnes App — UI Redesign v2.0

**Дата:** 2026-04-19  
**Статус:** Approved  
**Подход:** A — Дизайн-система → экраны (дизайн-система сначала, потом каждый экран)

---

## 1. Контекст

В проект добавлена папка `fitnes-design/` с handoff-пакетом от Claude Design: интерактивный прототип `Fitnes Redesign.html` с полной дизайн-системой и 5 экранами. Задача — обновить визуал всего приложения по новой дизайн-системе, не трогая работающий функционал (Supabase, TanStack Query, Auth, Camera, HealthKit).

## 3. Экраны

### 3.1 Home (`app/(tabs)/index.tsx`)

**Что меняется:**
1. **Анимированное приветствие** — "С возвращением 👋" (FadeOut, 0.8с) → имя пользователя из `profile.full_name` (FadeIn, 0.8с, задержка 1.5с). Реализация: `react-native-reanimated` FadeIn/FadeOut.
2. **3 метрики-плитки** (новый компонент) — ряд из 3 карточек: Ккал (оранжевый), Шаги (зелёный), Тренировки (синий). Данные из существующих источников.
3. **Hero-карточка тренировки** — градиентный фон (135deg, #1a4a6e → #2B8EF0), фото упражнения справа (Image компонент, placeholder до получения фото), кнопка "Начать" с backdrop-blur.
4. **Иконка воды** — SVG капля с бликом (синяя, #2B8EF0).
5. **Иконка шагов** — SVG кроссовок Nike-style (зелёный, #3DD87A, вариант B).
6. **КБЖУ сегодня** — заголовок меняется с "КБЖУ баланс" на "КБЖУ сегодня". Новый layout: макронутриенты в строку с цветными прогресс-барами.

**Что не трогаем:** данные воды/шагов из HealthKit, данные КБЖУ из Supabase, кнопка добавления еды.

### 3.2 Workouts (`app/(tabs)/workouts.tsx`)

**Что меняется:**
1. **Градиентные карточки тренировок** — каждая карточка получает свой градиент. Фото упражнения справа (когда будут предоставлены).
2. **Недельный стрип дней** — дни с запланированными тренировками отмечены зелёной точкой снизу. Активный день — синяя пилюля.
3. **Бейджи сложности** — обновлены цвета под новую систему.

**Что не трогаем:** данные из Supabase, клик → экран деталей, фильтрация по дням.

### 3.5 Profile (`app/(tabs)/profile.tsx`)

**Что меняется:**
1. **Плитки статистики** — компактный ряд из 3 карточек: Рост / Вес / Цель (вместо списка строк)
2. **Бар-чарт активности** (новый) — недельный столбчатый график. Данные: количество завершённых тренировок по дням недели из Supabase (`workouts` таблица, поле `completed_at`). Если данных нет — показываем нули. Пиковый день → синий (`#2B8EF0`), остальные → `rgba(43,142,240,0.25)`. Компонент: `react-native-chart-kit` BarChart.
3. **Переключатель тем** — обновлён стиль под новую систему, логика сохраняется.

**Что не трогаем:** данные профиля из Supabase, добавление веса, линейный график веса (остаётся ниже бар-чарта), переключатель тем (Light/Dark/System), кнопки выхода и повторного онбординга.

### 3.6 Welcome (`app/welcome.tsx`)

**Что меняется:**

- Активная точка пагинации → синяя, расширенная

**Что не трогаем:** 3 слайда, логика навигации.

### 3.7 Auth (`app/auth/login.tsx`, `register.tsx`)

**Что меняется:**

### 3.8 Onboarding (`app/onboarding/index.tsx`)

**Что не трогаем:** 23 параметра, логика сохранения, расчёт КБЖУ на последнем шаге.

## 4. Порядок реализации (Подход A)

1. **Шаг 1 — Шрифт и токены** — установить Manrope, обновить `tailwind.config.js`, `global.css`
2. **Шаг 2 — Атомарные компоненты** — Card, CustomButton, CustomInput, Badge
3. **Шаг 3 — Home** — метрики, анимация, Hero-карточка, SVG иконки, КБЖУ
4. **Шаг 4 — Workouts** — градиентные карточки, стрип дней
5. **Шаг 5 — Profile** — плитки статистики, бар-чарт
6. **Шаг 6 — Chat + Scanner** — цветовые обновления
7. **Шаг 7 — Welcome + Auth + Onboarding** — обновление остальных экранов
8. **Шаг 8 — Workout Detail** (`app/workouts/[id].tsx`) — обновить детальный экран тренировки под новую дизайн-систему (цвета, шрифты, компоненты)
